# Реестр игровых дисков

Веб-служба на ASP.NET Core — хранит в памяти процесса записи реестра игровых дисков (название игры, год выпуска) и предоставляет API для просмотра списка, получения по идентификатору и создания новых записей.

## Требования

- [.NET 11 SDK](https://dotnet.microsoft.com/download/dotnet/11.0)

## Запуск

### Сборка и запуск

Запустите проект в Visual Studio Code через терминал:
```
dotnet run
```

После запуска в консоли будет указан адрес приложения, например:

- **HTTP:** `http://localhost:54255`
- **HTTPS:** `https://localhost:54254`


---

## API

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/items` | Список всех записей |
| GET | `/api/items/{id}` | Одна запись по GUID |
| POST | `/api/items` | Создание записи (тело: `{"name": "...", "year": 2020}`) |

Формат элемента: `{ "id": "guid", "name": "string", "year": number }`.

Все ответы с ошибкой возвращают JSON вида: `{ "code": "...", "message": "...", "requestId": "..." }`. В заголовке ответа передаётся `X-Request-Id` — по нему можно найти запись в логах приложения.

## Испытания и проверка результата
1. **Логика предметной области** — unit-тесты валидатора (`CreateItemRequestValidatorTests`): непустое имя, неотрицательный год.
2. **Взаимодействие с веб-службой** — интеграционные тесты (`ItemsApiTests`): создание элемента и получение его по id; запрос по несуществующему id возвращает 404 в едином формате ошибки (`code`, `message`, `requestId`).
3. **Журнал** — для каждого запроса в консоли есть строка с идентификатором запроса и временем выполнения (мс).

Запуск всех тестов:

```bash
dotnet test task1_framework.sln
```

## Структура проекта

- **`task1_framework.sln`** — файл решения (открывать в IDE).
- **`Pr1.MinWebService.csproj`** — проект веб-приложения.
- **`Program.cs`** — конфигурация приложения и маршруты API.
- **`Domain/`** — модели (Item, CreateItemRequest, ErrorResponse).
- **`Services/`** — репозиторий в памяти, идентификатор запроса.
- **`Middlewares/`** — конвейер: RequestId, логирование/время, обработка ошибок.
- **`Errors/`** — типы исключений (Validation, NotFound).
- **`POSTMAN.md`** — подробная инструкция по проверке в Postman.

Данные хранятся в памяти процесса; при перезапуске реестр очищается.
